How the Algorithm Works

üìä Table Filling Phase

The algorithm builds a 2D table where each cell [i,j] represents the length of the longest common subsequence between the first i characters of String 1 and the first j characters of String 2.

  • If characters match: LCS[i,j] = LCS[i-1,j-1] + 1
  • If characters don't match: LCS[i,j] = max(LCS[i-1,j], LCS[i,j-1])

The bottom-right cell contains the length of the LCS.

üîô Backtracking Phase

Once the table is filled, we trace back from the bottom-right corner to find the actual subsequence:

  • If characters match: Include this character in the LCS and move diagonally (‚Üñ)
  • If value came from above: Move up (‚Üë)
  • If value came from left: Move left (‚Üê)

The characters collected during diagonal moves form the LCS.

‚ö° Complexity

  • Time: O(m √ó n) where m and n are the lengths of the two strings
  • Space: O(m √ó n) for the DP table

üí° Example

For strings "ABCD" and "ACBD":

  • The DP table fills left-to-right, top-to-bottom
  • Characters 'A', 'B', 'D' match in order ‚Üí LCS = "ABD" (length 3)
  • The backtracking path shows which cells contributed to the LCS
